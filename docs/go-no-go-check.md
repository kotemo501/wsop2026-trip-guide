# 出発GO/NO-GOカード

出発前日または出発当日の朝に、旅行代理店の最終確認のように見る1枚。公開版なので、予約番号、参加者名、チケットQR、保険証券番号、カード番号、個人の連絡先は書かない。

## 結論

全項目がGOなら出発準備は実務上OK。すぐ決まらないものはHOLDにして、担当、確認タイミング、代替手段が決まっていれば運用できる。NO-GOは、出発や安全に直結する確認ができない時だけにする。

## NO-GOなら先に潰す順番

1. パスポート、ESTA、ツアー最終案内、航空/ホテル関連。
2. Grand Canyon予約確認と05:05集合。
3. Sphere担当と6/27荷物/空港集合の確認タイミング。
4. 保険、カード停止、予備カード、Emergency現金。
5. Tag Team現地確認の動き方。
6. USD現金レンジと財布分け。
7. 紙控えとオフライン保存。

## 出発前日に2人で声に出すこと

- 予約番号、QR、保険証券番号、カード停止窓口はGitLabにはない。必要なら非公開控えを見る。
- 6/25は05:05 Grand Canyon pickupを絶対に外さない。
- 6/27はSphere後の荷物回収と空港集合を最優先にする。
- 疲れている日はポーカーより睡眠、水、食事を優先する。
- 別行動する時は、再集合場所と時刻を決める。

## まだ未確定でもGOにできる条件

| 未確定 | GOにする条件 |
| --- | --- |
| 空港集合の詳細が最終案内待ち | 最終案内の保存先、確認予定、公開版への更新先が決まっている |
| Sphere座席詳細 | 友人担当であること、確認タイミング、公開版への反映範囲が決まっている |
| Tag Team登録導線 | 現地確認前提で、6/23に導線確認、6/24 10:00に動くことが決まっている |
| dカード GOLD U付帯保険 | 補償確認中でも、足りない場合に追加保険を検討する期限と確認先が決まっている |
| USD現金の実額 | 固定額ではなく、基本レンジと用途別財布で運用することが決まっている |
| 保険の実番号 | 非公開控えと紙で見られ、公開版には実番号がない |
| カード停止窓口 | 非公開控えと紙で見られ、予備カードとEmergency現金が分散済み |
