# 公開用 最終案内書

旅行代理店の最終案内書のように、出発前から帰国までの見方を1枚にまとめる。GitLab Pagesで公開する前提なので、予約番号、参加者名、確認メール本文、チケットQR、保険証券番号、カード番号、個人の連絡先は書かない。

## このページの役割

- 友人にURLを送った後、最初に見てもらう全体案内。
- 出発前日に「準備が足りているか」を確認する案内。
- 現地で迷った時に、どのページを開くかを決める案内。

実番号やチケットはGitLab外の非公開控えで持つ。公開版は、時刻、場所、持ち物、支払い種別、判断ルールだけを見る。

## まず見る順番

1. [friend-quickstart.md](friend-quickstart.md) でURLを保存する。
2. [share-summary.md](share-summary.md) で旅の全体像を見る。
3. [concierge-handoff.md](concierge-handoff.md) で旅行代理店の担当者メモとして全体状態を見る。
4. [trip-status-dashboard.md](trip-status-dashboard.md) で予約済みと未完実務を分けて見る。
5. [go-no-go-check.md](go-no-go-check.md) で出発できる状態か判定する。
6. [daily-concierge.md](daily-concierge.md) を毎朝の入口にする。
7. [confirmed-bookings.md](confirmed-bookings.md) で予約済みの集合時刻と場所を見る。
8. [sos-quick-card.md](sos-quick-card.md) を緊急時の入口にする。
9. 余裕がある日は [fun-mission-card.md](fun-mission-card.md) で今日の楽しみを1つ選ぶ。

## 出発前日までに終わらせること

### 公開しおり

- トップページを2人ともスマホで開ける。
- トップ下部ナビの「今日」「添乗」「予約」「SOS」「旅程」を2人とも開ける。
- [today-command.md](today-command.md)、[daily-concierge.md](daily-concierge.md)、[confirmed-bookings.md](confirmed-bookings.md)、[sos-quick-card.md](sos-quick-card.md)、[pocket-itinerary.md](pocket-itinerary.md) をホーム画面かブックマークへ入れる。
- [fun-mission-card.md](fun-mission-card.md) も開ける。
- 一度オンラインで開き、オフラインでも読めるか確認する。

### 非公開控え

- Grand Canyon予約票/予約確認を各自のメール、Wallet、ローカル保存で開ける。
- Sphereチケットは購入後、公式アプリ/Walletで開ける。
- ツアー最終案内は、空港集合と緊急導線だけすぐ見られるようにする。
- 保険、カード停止窓口、証券番号、暗証番号はGitLab外で持つ。

### 紙

- [paper-backup-card.md](paper-backup-card.md) を印刷する。
- 実番号が必要なものはGitLabではなく、手書きまたは別紙の非公開控えにする。
- 2人で紙とEmergency現金を分散して持つ。

## 公開前の最終確認

- GitLabに予約票PDFがない。
- 予約番号、参加者名、確認メール本文、個人の電話/メールがない。
- チケットQR、バーコード、パスポート/ESTA番号、保険証券番号、カード番号がない。
- 公開しているのは時刻、場所、支払い種別、持ち物、判断ルールだけ。
- `./scripts/build_public.sh` と `./scripts/check_public.sh` を通す。
