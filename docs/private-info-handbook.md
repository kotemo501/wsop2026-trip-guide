# 非公開情報の持ち方

GitLabで公開するしおりと、予約番号・個人情報を入れる非公開控えを分けるための運用メモ。これは公開してよい説明書で、実際の番号や氏名は書かない。

## 基本ルール

| ルール | 理由 |
| --- | --- |
| 公開版には行動に必要な情報だけ書く | 集合時刻、集合場所、支払い種別、持ち物が分かれば現地で動ける |
| 予約番号、氏名、証券番号、カード情報は書かない | GitLab PagesやIssueに出ると回収できない前提で扱う |
| 非公開控えは2人とも見られる形にする | 片方のスマホ紛失や電池切れでも詰まない |
| 紙控えを1セット持つ | スマホ紛失、通信不良、アプリ障害への保険 |
| 更新したら公開版へ「公開できる要点」だけ反映する | 非公開情報なしでもしおりが使える状態を保つ |

## 置き場所の分担

| 情報 | 公開版 | 非公開控え | 紙 |
| --- | --- | --- | --- |
| Grand Canyon集合時刻・場所 | 書く | 書いてよい | 書く |
| Grand Canyon予約番号 | 書かない | 書く | 書く |
| 予約票PDF/QR | 置かない | 保存 | 印刷またはスクショ |
| Sphere公演名・時刻・座席エリア | 書く | 書いてよい | 書く |

| パスポート番号 | 書かない | 書く | コピーは各自判断 |
| ESTA番号 | 書かない | 書く | 書く |
| 海外旅行保険の証券番号 | 書かない | 書く | 書く |
| クレカ停止窓口 | 書かない | 書く | 書く |
| ツアー緊急連絡先 | 公開してよい一般窓口だけ | 最終案内の番号を書く | 書く |

## ローカル保存の作り方

推奨:

1. リポジトリ直下ではなく、各自のスマホ/クラウド/ローカルに非公開フォルダを作る。
2. どうしてもこの作業ディレクトリ内に置くなら、`.gitignore`済みの`local-private/`または`private/`だけに置く。
3. ファイル名に予約番号や氏名を入れない。
4. スクショには不要なメール本文や住所が写らないようにする。
5. GitLab IssueやMR本文へ貼らない。

非公開フォルダ例:

```text
local-private/
  grand-canyon/
    reservation-note.md
    ticket-screenshot.png
  sphere/
    mobile-ticket-note.md
  insurance/
    emergency-note.md
  cards/
    stop-contact-note.md
```

この例はフォルダ構成だけ。実ファイルはGitLabへコミットしない。

## 公開版へ反映してよい書き方

| 非公開で持つ内容 | 公開版の書き方 |
| --- | --- |
| Grand Canyon予約番号 | `予約番号は非公開。各自のメール/Wallet/ローカル保存で確認` |
| Sphere座席番号 | `座席は購入後にエリアだけ反映。チケット詳細は非公開` |
| 保険証券番号 | `海外旅行保険は加入状況と緊急時の確認手順だけ反映` |
| カード停止窓口 | `カード停止窓口は非公開控えと紙へ保存` |
| ツアー最終案内の緊急番号 | `ツアー緊急連絡先は最終案内の非公開控えで確認` |

## 更新フロー

1. 予約・購入・確認をする。
2. [confirmation-intake-form.md](confirmation-intake-form.md) で公開版に書く要点を分ける。
3. 非公開控えに番号、QR、氏名、証券番号などを保存する。
4. 公開版には時刻、場所、持ち物、支払い方法、判断ルールだけ反映する。
5. [publish-checklist.md](publish-checklist.md) のチェックを通す。
6. 2人ともスマホで公開版と非公開控えを開けるか確認する。
