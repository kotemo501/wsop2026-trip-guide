# スマホセットアップ手順

出発前に、GitLab公開しおりと非公開控えをスマホで使える状態にする。予約番号、チケットQR、保険証券番号、カード情報はこの公開しおりには入れない。

## 目的

- 現地で通信が弱くても、集合・持ち物・支払い・SOS手順を開ける。
- 片方のスマホが使えなくても、もう片方が同じ情報を見られる。
- GitLab Pagesには公開できる情報だけを置き、予約票や個人情報は別で持つ。

## 1. 公開しおりを入れる

| 入れるもの | やること |
| --- | --- |
| トップページ | ブラウザで開き、ホーム画面追加またはブックマーク |
| [今日の司令室](today-command.md) | 朝起きて最初に開く |
| [予約済み一覧](confirmed-bookings.md) | 集合場所、集合時刻、追加支払いを確認 |
| [共有用1枚サマリー](share-summary.md) | 友人に送るURLとして保存 |
| [現地ポケット旅程表](pocket-itinerary.md) | 朝と移動中にすぐ開ける場所へ |
| [60秒SOSカード](sos-quick-card.md) | 焦った時に最初に開く |
| [日別コンシェルジュ](daily-concierge.md) | 朝、移動前、寝る前の添乗メモとして保存 |
| [旅のミッションカード](fun-mission-card.md) | 余裕がある日に楽しみを1つ選ぶ |

確認:

- トップページの下部ナビで「今日」「予約」「SOS」「旅程」を開ける。
- 資料リーダーで上のMarkdownを開ける。
- 1枚サマリーのURLを相方に送れる。

## 2. 非公開控えを入れる

GitLab外に保存する。

| 非公開で持つもの | 保存先の例 |
| --- | --- |
| Grand Canyon予約番号、予約票、予約確認 | メール、Wallet、ローカル保存、紙 |
| SphereチケットQR/バーコード | 公式アプリ、Wallet、スクショ可否確認後の保存 |
| ツアー最終案内、空港集合情報 | メール、ローカル保存、紙 |
| 保険証券番号、緊急窓口 | 保険アプリ、非公開メモ、紙 |
| カード停止窓口 | カード会社アプリ、非公開メモ、紙 |

ファイル名に予約番号や氏名を入れない。

## 3. オフライン確認

出発前に機内モードで確認する。

1. GitLab Pagesのトップを開く。
2. [today-command.md](today-command.md) を開く。
3. [confirmed-bookings.md](confirmed-bookings.md) を開く。
4. [sos-quick-card.md](sos-quick-card.md) を開く。
5. [pocket-itinerary.md](pocket-itinerary.md) を開く。
6. [daily-concierge.md](daily-concierge.md) を開く。
7. Grand Canyon予約票とSphereチケットをGitLab外から開く。
8. 保険窓口とカード停止窓口をGitLab外から開く。

開けないものがあれば、スクショ、PDF保存、Wallet、紙のどれかで補完する。

## 4. ホーム画面に置く順番

| 優先 | 置くもの | 用途 |
| --- | --- | --- |
| 1 | GitLab Pagesトップ | 全体入口 |
| 2 | 今日の司令室 | 朝の3アクション |
| 3 | 日別コンシェルジュ | 朝、移動前、寝る前の添乗メモ |
| 4 | 予約済み一覧 | 集合、支払い、予約状況 |
| 5 | 60秒SOSカード | 遅刻、紛失、体調不良 |
| 6 | ポケット旅程表 | 今日の動き |
| 7 | 非公開控え | 予約番号、QR、保険、カード |

## 5. 出発前日の合格ライン

詳しい最終判定は [departure-readiness.md](departure-readiness.md) を見る。

- 2人ともトップページをスマホで開ける。
- 2人ともGrand Canyon予約票をGitLab外で開ける。
- 2人ともSphereチケットをGitLab外で開ける。
- 保険、カード停止、ツアー緊急連絡先はスマホと紙で確認できる。
- GitLabに予約票PDF、予約番号、氏名、QR、証券番号、カード番号を置いていない。
