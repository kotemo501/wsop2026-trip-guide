## GO/HOLDの考え方

| 判定 | 使う場面 |
| --- | --- |
| GO | 予約、持ち物、非公開控え、担当がそろっている |
| HOLD | 未確定はあるが、担当、確認タイミング、代替手段、更新先が決まっている |
| NO-GO | パスポート、予約確認、安全導線、支払い手段が確認できない |

## 友人に送る時の文章

```text
WSOP 2026の公開用しおりです。
予約番号やチケットQRは載せていないので、URLをスマホのホーム画面に追加してください。
最初に「友人向けクイックスタート」→「1枚サマリー」→「ポケット旅程表」の順で見てください。
予約票、保険、カード停止窓口、チケットQRはGitLab外の非公開控えで持ちます。
```

送る場面別の文面は [share-message-templates.md](share-message-templates.md) を使う。

## 公開版の守り方

- GitLabに予約票PDFを置かない。
- Issueやコメントにも予約番号、個人名、確認メール本文を書かない。
- チケットQRやバーコードはスクショ名にも入れない。
- 集合時刻、集合場所、持ち物、支払い種別、判断ルールだけ公開版へ転記する。
