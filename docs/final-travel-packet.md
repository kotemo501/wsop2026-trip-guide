# 最終トラベルパック

## 目的

- スマホが圏外でも、集合・移動・支払いで困らない状態にする。
- 片方のスマホ紛失や電池切れでも、もう片方が動ける状態にする。
- GitLab Pagesには公開できる要点だけ残し、実番号は非公開控えで持つ。

| 紙の最小セット | パスポート周辺、Grand Canyon、保険、カード停止、空港集合 | 2人で分散して持つ |

## 紙で持つもの

紙は全部を印刷しない。番号が必要なものと、スマホ紛失時に動くための最小限にする。

| 紙 | 内容 |
| --- | --- |
| 1枚目 | 日別ミニ旅程、ホテル、主要集合時刻 |
| 2枚目 | Grand Canyon集合場所、05:05 pickup、予約会社、追加fee。予約番号は公開版に印字しない |
| 3枚目 | 6/27 Sphere、荷物回収、空港集合時刻・場所 |
| 4枚目 | 保険、カード停止窓口、ツアー緊急連絡先 |
| 各自 | パスポート/ESTAまわりの控え。番号は公開版に載せない |

公開してよい行動情報だけを印刷するなら [paper-backup-card.md](paper-backup-card.md) を使う。予約番号、証券番号、個別連絡先などの実番号は、印刷後に手書きするか、別の非公開紙で持つ。

## 印刷しないもの

| 印刷しない | 理由 |
| --- | --- |
| GitLabに置いた全資料 | 枚数が多く、現地で探しにくい |
| 予約確認メール全文 | 個人情報が多い。必要項目だけ非公開控えへ |
| クレカ番号全面 | 紙の紛失リスクが高い。停止窓口とカード会社名を優先 |
| 大量のスクショ | 必要な画面だけ。ファイル名に予約番号や氏名を入れない |
