# 友人へ送るメッセージ集

LINE、Slack、メールでそのまま送れる公開用文面。予約番号、参加者名、チケットQR、保険証券番号、カード番号、個人の連絡先は書かない。

## 最初にURLを送る

```text
WSOP 2026ラスベガス旅の公開しおりです。
スマホで見る前提で、日程、予約集合、持ち物、SOS、予算、現地メモをまとめています。

まずこの順で見てください。
1. トップページ
2. 友人向けクイックスタート
3. 公開用 最終案内書
4. 共有用1枚サマリー

予約番号、チケットQR、保険/カード情報はGitLabには載せていません。
必要なものは各自のメール、Wallet、非公開メモ、紙で持つ運用です。
未確定のものはGO/HOLD/NO-GOで管理します。担当と確認タイミングが決まっているものはHOLDとして進めます。
```

## 出発前に保存してもらう

```text
出発前にスマホへ保存しておいてほしいページです。

- トップページ
- 今日の司令室
- 日別コンシェルジュ
- ポケット旅程表
- 予約済み一覧
- 60秒SOSカード

一度オンラインで開いて、機内モードでも読めるか確認してください。
予約票、Sphereチケット、保険、カード停止窓口はGitLab外の非公開控えで持ちます。
```

## 現地で朝に送る

```text
今日の動きはまず「今日の司令室」と「日別コンシェルジュ」を見てください。
固定時刻、持ち物、やらないことだけ先に確認します。

体力が微妙なら「体力・休憩カード」を優先。
余裕があれば「旅のミッションカード」から小さな楽しみを1つ選びます。
```

## 未決事項を相談する

```text
未決事項は「決定管制表」と「GO/NO-GOカード」を見ながら相談したいです。

いまの扱いはこの形です。
1. Sphereチケット: 友人担当待ち。確定後は座席エリアと入場時刻だけ公開版へ反映
2. 6/27荷物預けと空港集合: ツアー最終案内待ち
3. Tag Team登録導線: 現地確認前提。6/23に会場導線、6/24は10:00目安
4. 保険とカード停止導線: dカード GOLD U付帯を確認し、足りなければ追加保険検討
5. USD現金とカード構成: 固定額ではなく基本レンジで可変運用

決まったら、公開版には時刻、場所、持ち物、支払い種別だけ反映します。
予約番号や個人情報はGitLab外に置きます。
```

## 緊急時に短く送る

```text
まず安全な場所に移動してください。
次に「60秒SOSカード」を開いてください。

今いる場所、何が起きたか、次にどこへ向かうかを共有してください。
命、事故、犯罪は911です。
予約番号やカード停止情報はGitLabではなく非公開控えを見ます。
```

## 最終日に送る
