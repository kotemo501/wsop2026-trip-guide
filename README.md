# WSOP 2026 ラスベガス旅のしおり

友人2人で「世界のヨコサワ WSOP 2026 ツアー」に参加する前提の公開用しおりです。GitLabでIssueやMerge Requestを使って、予定・予算・候補を更新していくための土台です。

## 旅の前提

| 項目 | 内容 |
| --- | --- |
| ツアー | ZIPAIRチャーター直行便で行くラスベガス8日間ツアー |
| 出発 | 2026-06-22(月) 成田発、ラスベガス着 |
| 滞在 | ホースシュー・ラスベガス 5連泊 |
| 現地最終日 | 2026-06-27(土) 夜に空港へ |
| 帰国 | 2026-06-29(月) 朝 成田着 |
| 主目的 | WSOP体験、低めのトーナメント、キャッシュゲーム、観光 |

## 今回の基本方針

- WSOP公式イベントは `$1,000 Mystery Millions` と `$1,000 Tag Team` を最優先候補にする。
- 高額イベントは見学・雰囲気重視にして、実プレイは低めのWSOP本戦、Daily Deepstack、キャッシュゲームを組み合わせる。
- 6/26(金) 19:00のWSOP公認フリーロールはツアー限定体験として原則参加候補にする。
- 最終日6/27(土)は夜に空港移動があるため、重い予定を入れすぎない。

## まず決めること

| 優先度 | 決めること | 期限目安 | メモ |
| --- | --- | --- | --- |
| 高 | 2人のポーカー予算上限 | 予約後すぐ | トーナメント、キャッシュ、食費とは別枠で管理 |
| 高 | WSOP本戦に何本出るか | 2026-05末 | 複数Dayイベントは翌日以降の予定を圧迫する |
| 高 | 6/24 Tag Teamに出るか | 2026-05末 | 2人参加なら記念度が高い |
| 高 | 6/26フリーロール参加 | 2026-06上旬 | 同日昼の予定を軽めにする |
| 中 | キャッシュゲームの上限・撤退ルール | 出発前 | 例: 1日最大損失、終了時刻、飲酒時のルール |
| 中 | 観光の本命を1-2個に絞る | 出発前 | ポーカー予定のブレに対応しやすくする |
| 中 | ESTA、パスポート、海外保険、通信 | 2026-05中 | ESTAは必須。パスポート情報はツアー手配にも必要 |

## GitLabでの運用案

- Issue: 「6/24 Tag Teamに出るか」「観光候補を決める」「1日ごとの予算上限」など、決める単位で立てる。
- ラベル: `decision`, `poker`, `tourism`, `budget`, `booking`, `todo`
- Merge Request: 予定を変えるときは `docs/schedule.md` を更新して、理由をMR本文に残す。
- Milestone: `出発前`, `現地1日目`, `現地中`, `帰国後`
